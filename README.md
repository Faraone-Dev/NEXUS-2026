## рҹҺҜ What It Does

NEXUS is a fully autonomous Solana trading system. It's not a signal channel вҖ” it's an end-to-end pipeline that:

1. **Scans** вҖ” Continuously discovers new tokens via DexScreener, Birdeye, PumpFun
2. **Validates** вҖ” Multi-layer rug detection (contract, liquidity, holders, on-chain)
3. **Analyzes** вҖ” 4 specialized DeepSeek AI agents + Aarna MCP technical analysis (44 indicators)
4. **Executes** вҖ” Auto-trades via Jupiter with position sizing and SL/TP management
5. **Learns** вҖ” Every trade outcome feeds back into reactive weight tuning
6. **Alerts** вҖ” Real-time Telegram notifications with P&L tracking

## рҹ§  Multi-Agent AI System

4 specialized DeepSeek agents work in sequence:

### Agent #1: Token Analyzer
```
Input:  Contract address, on-chain data, TA signals
Output: Security score 0-100, red flags list
Focus:  "Is this token safe?"
```

### Agent #2: Sentiment Analyzer
```
Input:  Volume trend, holder growth, social mentions
Output: Hype score 0-100, trending direction
Focus:  "Is there momentum?"
```

### Agent #3: Risk Assessor
```
Input:  Holder distribution, LP status, dev wallet activity
Output: Rug probability %, risk level (LOW/MED/HIGH)
Focus:  "What can go wrong?"
```

### Agent #4: Master Decision
```
Input:  Results from Agent 1-3 + Aarna TA (44 indicators)
Output: BUY/SELL/WAIT, entry price, SL, TP, confidence %
Focus:  "Should we trade?"
```

**Decision Thresholds:**
- вң… Score вүҘ 65 + Confidence вүҘ 70% вҶ’ Execute trade
- вҡ пёҸ Score 50-65 вҶ’ Alert only (no auto-trade)
- вқҢ Score < 50 вҶ’ Skip

## рҹ“Ў Aarna MCP Integration

Technical analysis powered by [Aarna ATARS](https://smithery.ai/servers/atars-MCP/aarnaai) via Smithery Connect:

- **44 indicators** per token (RSI, MACD, Bollinger Bands, EMA cross, ADX, Stochastic, etc.)
- **Composite signal** вҖ” BULLISH / BEARISH / NEUTRAL with bullish/bearish breakdown counts
- **Transport** вҖ” JSON-RPC over HTTP via Smithery Connect API
- **Integration** вҖ” Feeds directly into Master Decision agent for TA-aware trade sizing

## рҹ“Ҳ Learning System

```
  Trade Executed
       в”Ӯ
       в–ј
  Wait for outcome (TP / SL / manual close)
       в”Ӯ
       в–ј
  ReactiveAI updates per-signal weights:
  вҖў Which data sources predicted correctly?
  вҖў Increase weight for winning signals
  вҖў Decrease weight for losing signals
       в”Ӯ
       в–ј
  Every 24h вҖ” LearningEngine (DeepSeek):
  вҖў AI post-mortem: "Why did trades win/lose?"
  вҖў Auto-adjust thresholds, SL/TP, sizing
  вҖў Insights saved to database
       в”Ӯ
       в–ј
  Next scan cycle uses updated weights
```
